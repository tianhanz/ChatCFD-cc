This OpenFOAM case employs the `simpleFoam` solver to simulate steady-state, incompressible turbulent flow through a duct with a 90-degree bend. The fluid is modeled as a Newtonian fluid with a constant kinematic viscosity (`nu = 1.5e-05 m²/s`). Turbulence is resolved using the `kEpsilon` RANS model, with both `k` (turbulent kinetic energy) and `epsilon` (turbulent dissipation rate) equations solved. 

Discretization schemes are primarily linear: Gauss linear for gradients and Gauss linear corrected for Laplacians. The convection term for momentum (`div(phi,U)`) uses the `Gauss limitedLinearV 1` scheme, while turbulence variables (`k`, `epsilon`) use the `Gauss limitedLinear 1` scheme for boundedness. Pressure-velocity coupling is handled via the SIMPLE algorithm with consistent formulation enabled (`consistent yes`) and no non-orthogonal corrections (`nNonOrthogonalCorrectors 0`). The pressure (`p`) equation is solved using a GAMG algebraic multigrid solver with Gauss-Seidel smoother and tight tolerance. Velocity (`U`), `k`, and `epsilon` equations use a smoothSolver with Gauss-Seidel smoother and relaxation factors applied (`U: 0.9`, `k: 0.7`, `epsilon: 0.7`) for stability.

Boundary conditions include a uniform inlet velocity (`U = (10 0 0) m/s`) with specified turbulent kinetic energy (`k`) based on 5% intensity and dissipation rate (`epsilon`) scaled by a mixing length. Walls apply a no-slip condition for velocity and standard wall functions (`nutkWallFunction`, `kqRWallFunction`, `epsilonWallFunction`) for turbulence parameters. The outlet uses a fixed pressure value and an `inletOutlet` condition for turbulence variables to prevent backflow issues. Initial fields are set uniformly: velocity `(0 0 0)`, pressure `0`, `k=1`, `epsilon=200`, `nut=0`.
